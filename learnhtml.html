HTML for beginners

Introduction

HTML is a very useful language for creating dynamic web pages. To create one, simply open up a text editor (I use CotEditor, but there are many different ones out there). You will then need to save your file as a .html document, then go to your web browser and click open page. You will be able to see what your code produces on the screen.

Starting your page

All html pages start like this:
<!DOCTYPE html>
<html>
Anything within these triangular brackets are known as tags . Don't forget to close your html tag at the end of your page!
</html>
You will need to do this with most of your tags.

Sectioning and organising your code

It is good practice to split your code up with tags. Some tags, such as headings, make a difference visually to the page, others, such as divs, just makes it easier for both you and someone else to read. It will also be very useful when we introduce styling later on.
Here are some common tags:

Paragraphs

Used for blocks of text of any length. It will create a blank line before and after the block, as a paragraph in a book would look.

<p>
This is a paragraph with multiple lines,
but will be displayed as one line on the web page.
</p>

This is a paragraph with multiple lines, but will be displayed as one line on the web page.

Code

Displaying code on a webpage isn't that simple, as it is simply read as code and displayed as it should be. Wrapping your code in the code tag changes the font and helps to group you code, but is not enough on its own. To display a triangular bracket we use '&lt' and '&gt'

<code> This is some code </code>

Pre

This is a useful tag which is used for displaying code but will take into account any line breaks and spaces within the text. 


<pre>
This is
a pre section of     text
</pre>
produces: 
This is
a pre section of     text

Horizontal Line

Produces a horizontal line, like above the 'Sectioning your code' heading on this page. This is an empty or self closing tag, which means it has no end tag.

<hr>

Line Break

Starts a new line. You can put as many as you like in a row to skip multiple lines. This is also empty, so doesn't need an end tag.

<br>

Span

Span has no visible difference on the page, but is useful for identifying specific words or parts of a paragraph. It will keep the text inline with the rest of the text, unlike paragraph.

<p> My name is Hannah. <span> I am 24 years old. </span> I am from Cambridge </p>

Spacing

It is good practice, and makes your code easier to read, to lay it out as follows. If it is a short snippet of code, you can keep it all on one line:
<p> This is a short sentance. </p>

For longer sections, it is standard to nest your code with a two space indent. You can nest further tags within these sections: 

<p>
  This is a paragraph of text with some
  <code>
    section of code
  </code>
  nested within it.
</p>

Div

Divs again have no visual effect on the text, but are useful for grouping and sectioning your code, and will be essential for styling with CSS later on. They can contain any number of other tags within them. For example, if you were writing a web page about yourself, you might have one div about your childhood, another about your school life, and another about your hobbies.

<div> This is a div </div>

id's and classes

id

id's can be applied to almost any element and are very useful for styling in CSS, as well as adding effects to objects like buttons in Javascript. Code can end up being very long and it's very useful to be able to refer back to specific headings or sections later down the line without having to scroll through the whole page. You should give it a name that is easy to remember relates to the section. id's are unique, only one element can have a specific id. They are added to tags like this:


<h1 id="first">
<div id="hobbies">
Properties that go inside the tag like this are known as attributes .

Classes

Classes are similar to id's but multiple tags can have the same class. For example if your web page had a section for several different people, you might want all of your sections about their family to have the same class.
<div id="person1" class="family">
<div id="person2" class="family">
You can assign multiple classes to the same tag:
<h1 class="hobbies lifestyle" >
Note that this means your class names should not include any spaces or they will be read as two different classes.

Lists

There are two types of lists, ordered and unordered. You could create your lists with just normal text, but it is much easier to keep track of your elements by using the list tags.

Unordered Lists

  <li> Second element </li>
  <li> Third element </li>
</ul>

  • Third element

Ordered Lists

You can also add a link to a website to a picture.

It is also a good idea to add an alt attribute to your image tag, this text will be shown if the image cannot be displayed on the page.

Videos

You can embed videos into your web page with the video tag. It is a good idea to add height and width attributes so your page doesn't flicker while the video is loading. 
 <video width="300" height="200" autoplay controls>
   <source src="movie.mp4" type="video/mp4">
 Video not supported.
The autoplay attribute will autoplay the video when the page loads, the controls attribute will display controls for the user. The text will only be displayed if the video cannot be loaded.

Tables

Our table will have three sections, the head, the body and the foot. This means you can edit the styling of these individually. When we start the table we can change the border size. 

<table border="5">
  <thead>
     <tr>
       <th> Shape </th>
       <th> Number of sides </th>
       <th> Number of Vertices </th>
      </tr>
  </thead>
  <tbody>
      <tr>
        <td> Triangle </td>
        <td colspan="2"> 3 </td>
      </tr>
      <tr>
        <td> Cube </td>
        <td> 6 </td>
        <td> 8 </td>
      </tr>
  </tbody>
  <tfoot>
      <td> Total </td>
      <td> 9 </td>
      <td> 11 </td>
  </tfoot>
</table>

As you can see, the colspan="2" means that number spans across two columns. You can do the same for rows with the atribute rowspan.

Forms

We may want the user to submit information to us, the way to do this is using form. It is important to label our forms so we can keep track of which information relates to which form.

Short text form

The simplest of forms may be used, for example, for a user to submit their name. 

<form>
  <label for="name"> Type your name here </label> <br>
  <input type="text" name="name" id="name" value="Joe Bloggs">
</form>


The label for attribute refers to the id of the input. It appears above the form. Screen reader uses will hear the label read out when focussed on this element.
The name attribute will help you match up data when it has been submitted.
The value attribute will automatically appear in the form before the user changes it.

Longer text form

You can set the maximum length you would like your user's response to be. 

<form>
  <label for="message">> Leave me a message </label> <br>
  <input type="text" name="message" id="message" rows="3" cols="40"> Your message here!
</form>

Password

If your wesbite requires people to log in, you can use the password type for your input. This will hide the text as you're typing it. Give it a go below. 

<form>
  <abel for="password"> Password </label> <br>
  <input name="password" id="password" type="password">
</form>

Number forms

These would be useful, for example, for your user to enter their age. Only numbers can be typed into this form. 

<form>
  <label for="age"> Age </label> <br>
  <input name="age" type="number" step="1" id="age"> 
</form>

Range forms

This allows the user to answer in the form of a scale. You can choose descriptions for each end of the scale and how many steps in between you would like. 

<form>
  <label for="hunger"> How hungry are you? </label> <br>
  <span> Famished </span>
  <input id="hunger" name="hunger" type="range" min="0" max="10" step="1">
  <span> Stuffed </span>
</form>

Famished Stuffed

Radio forms

Radio forms give just two options, and one must be selected. They are useful for yes/no or on/off type questions. 

<form>
  Do you like dogs?
  <input type="radio" name="dogs" id="yes" value="yes"> 
  <label for="yes"> Yes </label>
  <input type="radio" name="dogs" id="no" value="no">
  <label for="no"> No </label>
</form>
Do you like dogs?

The value attribute is what will be submitted to the person recieving the form

Datalist

These forms give several options, but also allows the user to type in their own answer. They can choose only one. 

<form>
  <label for="breed"> What is your favourite breed? </label>
  <input list="breeds" id="breed" name="breed">
  <datalist id="breeds">
    <option value="German shepherd"> </option>
    <option value="Husky"> </option>
    <option value="Spaniel">  </option>
  </datalist>
</form>

Select

Here the user must select one of the options given. 

<form>
  <label for="number"> How many dogs do you have? </label>
  <select id="number" name="number">
    <option value="0"> 0 </option>
    <option value="1"> 1 </option>
    <option value="2+"> 2+ </option>
  </select>
</form>

The text in between the option brackets is what will appear in the drop down list.

Checkbox

This type of form allows the user to select zero or multiple options from a list. 

<form>
  What colour of dogs do you like? <br>
  <input type="checkbox" id="colour1" name="colour1" value="Black">
  <label for = "colour1"> Black </label> <br>
  <input type="checkbox" id="colour2" name="colour2" value="Brown">
  <label for = "colour2"> Brown </label> <br>
  <input type="checkbox" id="colour3" name="colour3" value="Blonde">
  <label for = "colour3"> Blonde </label> <br>
  <input type="checkbox" id="colour4" name="colour4" value="Other">
  <label for = "colour4"> Other </label> <br>
</form>
What colour of dogs do you like?




Submit

There wouldn't be much point filling in these forms if we can't submit them. 

<form>
  <input type="submit" value="Submit">
</form>
